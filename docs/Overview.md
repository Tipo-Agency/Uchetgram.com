# Общее техническое задание: Uchetgram

## 1. Общая информация

### 1.1. Назначение системы

Uchetgram — это **SaaS‑платформа** для управления бизнесом в сфере услуг. Система построена на концепции **универсальной сущности**: одна базовая модель данных может быть задачей, лидом, финансовой заявкой, документом или шагом бизнес‑процесса.

Платформа подходит для салонов красоты, клиник, фитнес‑клубов, автосервисов, ресторанов, ритейла и других отраслей.

### 1.2. Основные возможности

- Управление задачами и проектами
- CRM: клиенты, сделки, воронки продаж
- Финансовый учёт: операции, планирование, заявки на выплаты
- Документы и файлы (внутренние, внешние ссылки)
- Бизнес‑процессы и автоматизация
- Складской учёт
- Аналитика и отчётность
- Интеграция с Telegram‑ботом
- Маркетплейс готовых отраслевых решений

## 2. Архитектура системы

### 2.1. Компоненты

- **Frontend** — React + TypeScript (Vite), маркетинговый сайт и веб‑приложение
- **Backend** — Python (FastAPI), REST API
- **База данных** — PostgreSQL
- **Telegram Bot** — Python (aiogram), работает через API backend
- **Admin Panel** — Super Admin для управления tenant’ами и платформой

### 2.2. Модули приложения

1. **Dashboard** — рабочий стол, сводка, быстрые действия
2. **Sales (CRM)** — воронка продаж, клиенты, сделки
3. **Finance** — финансы, операции, заявки, планирование
4. **Documents** — документы, папки, файлы
5. **Tasks** — задачи, канбан, список
6. **Processes** — бизнес‑процессы (BPM)
7. **Analytics** — отчёты и дашборды
8. **Inventory** — склад и запасы
9. **Marketplace** — готовые решения и конфигурации
10. **Settings** — настройки команды, интеграций, биллинга

## 3. Пользователи и роли

### 3.1. Уровень SaaS

- **Super Admin** — доступ к админке платформы (tenant’ы, пользователи, биллинг)
- **Tenant Owner** — владелец организации, полный доступ внутри своей организации

### 3.2. Уровень организации (tenant)

- **Admin** — администратор организации, управление пользователями и настройками
- **Employee** — сотрудник с ограниченным доступом

### 3.3. Права доступа

- Super Admin может управлять всеми организациями
- Admin может управлять пользователями и настройками своей организации
- Employee создаёт и редактирует свои задачи, работает с клиентами и сделками в рамках назначений

## 4. Универсальная сущность

В центре системы — одна базовая модель, которая в разных контекстах становится:

- **Задача** — с дедлайном, ответственным, чек‑листом
- **Лид / Сделка** — с суммой, этапом воронки, контактом
- **Финансовая заявка** — с суммой, категорией, статусом согласования
- **Транзакция** — движение денег (банк, 1С)
- **Документ** — файл, ссылка или внутренний текст
- **Бизнес‑процесс** — цепочка связанных задач с ветвлением
- **Сообщение** — контент и канал связи

Это позволяет использовать единые компоненты отображения, поиска и фильтрации по всей системе.

## 5. Основные функции

### 5.1. Задачи

- Создание, редактирование, удаление
- Назначение исполнителей
- Приоритеты и статусы
- Комментарии и вложения
- Представления: таблица, канбан

### 5.2. CRM

- Клиенты и контакты
- Воронки продаж
- Сделки и договоры
- Дебиторская задолженность

### 5.3. Финансы

- Операции (импорт банка, ручной ввод)
- Планирование и бюджеты
- Заявки на выплаты с согласованием
- Отчёты P&L, план‑факт

### 5.4. Документы

- Папки и файлы
- Внешние ссылки (Figma, Google Sheets)
- Внутренние текстовые документы

### 5.5. Бизнес‑процессы

- Шаги с назначенными исполнителями
- Ветвление (например, одобрить/отклонить)
- Автоматическое создание задач при переходе шага

### 5.6. Уведомления

- Лента активности (activity log)
- Telegram‑уведомления по настройкам пользователя

## 6. Интеграции

### 6.1. Telegram‑бот

- Авторизация пользователей
- Быстрые команды: задачи, заявки, отчёты
- Уведомления о событиях

### 6.2. Внешние сервисы (планируется)

- Банки (выписки)
- 1С
- Instagram, сайты (лиды)

## 7. Безопасность

### 7.1. Аутентификация

- JWT или сессионные токены
- Пароли хешируются (bcrypt)
- Двухфакторная аутентификация (опционально)

### 7.2. Авторизация

- Роли и права доступа
- Изоляция данных по tenant’ам (multi‑tenancy)

### 7.3. Защита данных

- HTTPS
- Чувствительные данные не хранятся в открытом виде
- Аудит критичных действий

## 8. Развёртывание

### 8.1. Frontend

- Сборка Vite
- Деплой на Nginx или CDN

### 8.2. Backend

- Uvicorn / Gunicorn
- systemd или Docker

### 8.3. Telegram Bot

- Отдельный процесс
- systemd или Docker

## 9. Технические требования

### 9.1. Браузеры

- Chrome, Edge, Firefox, Safari (последние версии)

### 9.2. Устройства

- Desktop — основная платформа
- Tablet — адаптивный дизайн
- Mobile — базовая поддержка

### 9.3. Производительность

- Загрузка страницы < 3 секунд
- Отклик на действия < 1 секунды
