# UI‑гайд по сущностям Uchetgram

## 1. Зачем этот документ

Цель: **зафиксировать визуальные паттерны** для всех сущностей, чтобы интерфейс был однородным, а новые экраны делались по тем же правилам.

Документ описывает:
- какие компоненты отвечают за каждую сущность;
- какие UI‑паттерны использовать (карточки, таблицы, табы, модалки);
- как выглядят заголовки, кнопки и фильтры.

---

## 2. Базовые паттерны

### 2.1. Заголовки страниц

- Компонент: `Header`.
- Показывает: название текущего раздела, поиск, уведомления, профиль пользователя.
- Названия модулей: Рабочий стол, CRM, Финансы, Документы, Задачи, Бизнес‑процессы, Аналитика, Склад, Маркетплейс, Настройки.

### 2.2. Навигация (Sidebar)

- Единственное левое меню приложения.
- Пункты: Рабочий стол, CRM, Финансы, Документы, Задачи, Процессы, Аналитика, Склад, Маркетплейс, Настройки.
- Внутри разделов — только табы или фильтры, без второго сайдбара.

### 2.3. Табы (вкладки)

- Стиль «пилюль»: скруглённые, `text-sm`, активный — тёмный фон, белый текст.
- Не больше одной строки; при избытке — горизонтальный скролл.
- Используются в: Dashboard (Входящие/Исходящие), Settings (разделы), модулях с подразделами.

### 2.4. Карточки

- Фон: `bg-white`, рамка `border border-border`, `rounded-xl`, `p-4` / `p-5`.
- Hover: лёгкая тень, подсветка рамки.
- Примеры: EntityCard, карточки клиентов, сделок, задач.

### 2.5. Таблицы

- Заголовок: `thead` с `bg-slate-50`.
- Строки с hover‑подсветкой.
- Действия (редактировать, удалить) — справа.

### 2.6. Модалки

- Затемнение: `fixed inset-0 bg-black/40 backdrop-blur-sm`.
- Карточка: `max-w-md` или `max-w-lg`, `rounded-xl`, `border`, `shadow-2xl`.
- Заголовок + кнопка закрытия (X).

---

## 3. EntityCard (универсальная карточка)

Один компонент для отображения любой сущности. Меняет:
- **Иконку** — по типу (task, deal, document, …).
- **Цветовую метку** — по приоритету или статусу.
- **Набор полей** — по типу (для задачи: дедлайн, исполнитель; для сделки: сумма, стадия).

Общие элементы:
- Заголовок (title)
- Статус (бейдж с цветом)
- Приоритет (если есть)
- Исполнитель (аватар или имя)
- Дата (создания, дедлайна — зависит от типа)
- Действия: открыть, редактировать (по hover или меню)

## 4. Сущности по модулям

### 4.1. Задачи (TasksModule)

- Список: карточки или таблица.
- Представления: Канбан (по статусам), Список.
- Карточка: название, статус, приоритет, исполнитель, дедлайн.
- Модалка: TaskModal для создания/редактирования.

### 4.2. CRM (SalesModule)

- Воронка: колонки = стадии, внутри — карточки сделок.
- Карточка сделки: название, сумма, стадия, клиент, ответственный.
- Список клиентов: карточки или таблица.

### 4.3. Финансы (FinanceModule)

- Табы: Обзор, Операции, Планирование, Заявки.
- Таблица операций: дата, сумма, тип, категория, описание.
- Карточки: баланс, P&L, план‑факт.

### 4.4. Документы (DocumentsModule)

- Дерево папок + список документов.
- Карточка документа: иконка по типу (file/link/internal), название, дата.
- Редактор: отдельный вид с полным экраном для внутренних документов.

### 4.5. Процессы (ProcessesModule)

- Список шаблонов процессов.
- Визуализация workflow: шаги и связи.
- Карточка экземпляра: процесс, текущий шаг, статус.

### 4.6. Аналитика (AnalyticsModule)

- Дашборд с карточками метрик и графиками (Recharts).
- Фильтры: период, категория.

---

## 5. Правила для новых экранов

1. Не добавлять новые боковые меню — только табы и фильтры внутри контента.
2. Табы везде в одном стиле (пилюли).
3. Карточки и таблицы — по описанным шаблонам.
4. Новые сущности привязывать к EntityCard или явно описать отличия.
5. Консистентность важнее «красоты» отдельного экрана.
