# Super Admin панель Uchetgram

## 1. Общая информация

### 1.1. Назначение

Super Admin — внутренний инструмент для управления SaaS‑платформой. Доступен только пользователям с ролью `super_admin`. Не предназначен для конечных пользователей продукта.

### 1.2. Размещение

- Отдельный маршрут `/admin` на основном домене или поддомен `admin.uchetgram.com`
- Отдельный layout: свой navbar, без Sidebar приложения
- Вход по тем же учётным данным, но проверка роли `super_admin`

## 2. Разделы

### 2.1. Dashboard (/admin)

- Сводка: количество tenant’ов, активных пользователей, выручка (из Stripe)
- Графики: регистрации за период, активность
- Последние tenant’ы, последние ошибки (если есть мониторинг)

### 2.2. Tenants (/admin/tenants)

- Таблица: название, slug, план, статус (активен/заблокирован), дата создания
- Действия: просмотр, редактирование, блокировка
- Создание tenant’а (ручное, для тестов)
- Фильтры: по плану, по статусу

### 2.3. Users (/admin/users)

- Поиск пользователя по email, имени
- Таблица: email, имя, tenant, роль, дата последнего входа
- Действия: просмотр, блокировка (если нужно)

### 2.4. Billing (/admin/billing)

- Сводка по Stripe: MRR, количество подписок по планам
- Ссылка на Stripe Dashboard для деталей
- При необходимости: ручная корректировка плана tenant’а

### 2.5. Settings (/admin/settings)

- Глобальные настройки: feature flags, лимиты, сообщения для пользователей
- Резервное копирование (ссылки на скрипты или статус)

## 3. API для админки

- Префикс: `/api/v1/admin`
- Все эндпоинты требуют роль `super_admin`

### 3.1. Tenants

- GET /api/v1/admin/tenants — список
- POST /api/v1/admin/tenants — создание
- GET /api/v1/admin/tenants/:id — детали
- PATCH /api/v1/admin/tenants/:id — обновление (план, is_active)

### 3.2. Stats

- GET /api/v1/admin/stats — метрики для дашборда

### 3.3. Users

- GET /api/v1/admin/users?q=... — поиск (с пагинацией)

## 4. Безопасность

- Строгая проверка роли на backend
- Логирование всех действий super_admin (кто, что, когда)
- Отдельный audit log (опционально)

## 5. UI

- Минималистичный: таблицы, формы, кнопки
- Можно использовать те же UI‑компоненты (Button, Card), но без тяжёлого дизайна продукта
- Мобильная версия — базовая (админка в основном с десктопа)
